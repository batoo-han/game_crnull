## Руководство администратора (скрытая админка)

### Доступ
Админка расположена по пути `/admin`.
Доступ защищён логином и паролем.

#### Дополнительная “маскировка” (опционально)
Можно включить дополнительный барьер:
- задайте `ADMIN_ROUTE_SECRET` в `.env`
- открывайте админку по URL `/admin/<секрет>`

Тогда бекенд будет отклонять любые `/api/admin/*` запросы без корректного секрета.
Это **не заменяет** логин/пароль, а дополняет их.

### Что можно настраивать
- Telegram (вкл/выкл, chat_id, шаблоны сообщений).
- Срок действия промокодов (TTL).
- Лимит выдач в сутки.
- Сложность бота.
- Параметры темы (цвета/интенсивность анимаций).

### Смена пароля администратора
В админке доступна форма смены пароля. Пароль в открытом виде в БД не хранится.

### Инициализация учётки администратора
Пароль администратора задаётся только на этапе первичной инициализации (из `.env`), а в базе хранится только хэш.

Подробный процесс запуска и инициализации: `docs/development_windows_venv.md` и `docs/deployment_ubuntu_docker.md`.
